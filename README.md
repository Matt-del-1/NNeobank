## NeoBank Core System(кредитование и транзакции)

Проект сделанный в рамках выполнения лабораторных работ по ПнаЯВУ(Java)

Система банковского обслуживания и управления кредитными линиями. Проект представляет собой бэкенд-платформу на базе Spring Boot для обработки финансовых операций, управления счетами клиентов и автоматизации начисления процентов.

## Технологический стек
* Язык программирования: Java 17
* Фреймворк: Spring Boot 3
* База данных: PostgreSQL
* ORM: Spring Data JPA (Hibernate)
* Тестирование: JUnit 5, Mockito, JMeter
* Инструменты: Docker, Docker Compose, Swagger/OpenAPI

## Предметная область (Entity Model)
В проекте реализованы следующие ключевые сущности и связи:
* User — Персональные данные клиента.
* Account — Банковские счета пользователя (One-to-Many).
* Transaction — Финансовые операции по счету (Many-to-One).
* Loan — Кредитные договоры, привязанные к счетам.
* PaymentSchedule — Графики платежей (One-to-Many от Loan).



## Статус выполнения лабораторных работ

### Лабораторная работа №1: Базовый REST-сервис
* Инициализация Spring Boot проекта.
* Разработка REST-контроллеров для управления пользователями.
* Настройка Checkstyle и приведение кода к единому стандарту.

### Лабораторная работа №2: Работа с данными (JPA/Hibernate)
* Проектирование схемы БД из 5+ сущностей.
* Настройка связей OneToMany, ManyToOne, ManyToMany.
* Демонстрация решения проблемы N+1 (EntityGraph/Fetch Join).
* Построение ER-диаграммы проекта.

### Лабораторная работа №3: Кэширование данных
* Реализация собственного in-memory кэша на основе HashMap.
* Организация доступа к данным по составному ключу.
* Механизмы инвалидации данных при их обновлении.

### Лабораторная работа №4: Обработка ошибок и логирование
* Глобальный обработчик исключений через ControllerAdvice.
* Использование Spring AOP для замера времени выполнения транзакций.
* Документирование API с помощью Swagger/OpenAPI.

### Лабораторная работа №5: Пакетная обработка и тестирование
* Пакетное начисление процентов по кредитам (Batch Processing).
* Использование Stream API и Optional в бизнес-логике.
* Unit-тестирование сервисного слоя с Mockito.

### Лабораторная работа №6: Многопоточность (Concurrency)
* Асинхронное выполнение операций через CompletableFuture.
* Демонстрация и устранение Race Condition при списании баланса.
* Нагрузочное тестирование системы в JMeter (50+ одновременных потоков).

### Лабораторная работа №7: Клиентская часть
* Разработка SPA-клиента для взаимодействия с API.
* Реализация интерфейсов для CRUD-операций и фильтрации данных.

### Лабораторная работа №8: Развертывание (Deploy)
* Контейнеризация приложения и базы данных через Docker Compose.
* Настройка CI/CD процесса в GitHub Actions (автоматическая сборка и тесты).

## Запуск проекта
1. Сборка проекта: ./mvnw clean install
2. Запуск приложения: ./mvnw spring-boot:run
